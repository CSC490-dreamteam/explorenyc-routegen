# Route Generation
This codebase is the main routing engine for ExploreNYC and the 2nd half of the route creation process. It receives the data from the golang codebase to run the combinatorics to decide on an optimal route for the enduser. This codebase is treated as a 'pure numbers' solver where it is given the bare minimum info needed to generate a time and cost efficient route. Once a route is created it is sent back to golang which polishes up the data.

This codebase hinges on the use of Google's CP-SAT solver. The main routing engine works by toggling on/off different possibiltiies of the route until it finds an optimal one. A route's optimality is scored by a scoring system that tracks penalities that weakens a route's quality. For example, as of writing this, the scoring system deems that if an optional stop adds over 20 minutes of travel to a trip, it isn't worth it and is dropped. If it only adds 10 minutes, it would be kept in as dropping a stop has a bigger penalty than traveling for 10 minutes. If there is too much idle time in the trip such as because of an appointment, the durations of activities are stretched out to cut down on idle time up to a cap.
